Abstract

Concise Research Summary

The project demonstrates a real-time video-processing architecture implemented entirely in hardware. It is suitable for a final-year presentation because the design story is clear, the results are measurable, and the visual output is reproducible.

Project Statement

This work implements a complete FPGA-based pipeline for capturing live camera data, transforming the pixel stream through a set of eight hardware filters, and presenting the processed image on a standard VGA monitor. The design uses the OV7670 camera sensor, pixel formatting logic, dual-port frame buffering, combinational filter circuits, and VGA timing generation. The output is evaluated through filter photographs, board references, and demonstration videos.

Acquisition OV7670 configuration and RGB565 capture through the camera interface.
Processing Eight real-time filters implemented directly in hardware.
Display VGA output with stable timing and on-screen frame delivery.

Methodology

Hardware Flow

The design sequence is presented as a compact pipeline rather than as a crowded block of text.

1

Camera setup

The OV7670 sensor is configured through SCCB so that the pixel stream is stable and synchronous.

2

Pixel capture

RGB565 data is sampled and converted into a usable internal format for downstream logic.

3

Frame buffering

Dual-port BRAM stores and forwards image data while keeping the read and write domains separate.

4

Filter bank

Eight combinational filters evaluate the same pixel stream and deliver distinct visual outcomes.

5

VGA output

The selected result is routed to a 640x480 display with consistent timing and frame stability.

Conclusion

Final Summary

The presentation closes with a concise account of what the design proves and where it can be extended.

Outcome

The project confirms that a compact FPGA system can acquire camera input, process it with multiple image filters, and present the result on a VGA monitor in real time. The result set and demonstration clips together provide a complete project showcase.

Possible Extension

  • Convert the prototype into a custom silicon implementation.
  • Extend the pipeline to higher resolutions and stronger timing margins.
  • Add controlled capture of more scenes for comparative evaluation.
