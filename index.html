I.

Abstract

This project presents the design and implementation of an FPGA-based real-time video processing pipeline using the OV7670 CMOS camera and the Basys 3 Artix-7 FPGA platform. The proposed system captures live video, buffers image data in dual-port Block RAM (BRAM), performs real-time image processing using lightweight combinational logic, and displays the processed output on a VGA monitor. To optimize memory utilization, the captured RGB565 pixel data is converted to RGB444 format before storage while maintaining satisfactory visual quality.

The architecture integrates camera configuration through the SCCB interface, frame buffering, VGA timing generation, and an eight-mode image filter bank controlled via a three-switch binary interface. The implemented filters include grayscale, color isolation, invert, sepia, posterize, and other real-time image enhancement modes. Since all filters are realized using purely combinational hardware, the system introduces negligible processing latency and supports seamless runtime filter switching without requiring FPGA reconfiguration.

A hardware-based FPS counter with on-screen display is incorporated to monitor real-time performance. Experimental implementation and validation using Xilinx Vivado demonstrate stable operation at a resolution of 320 × 240 pixels with a frame rate of approximately 30 FPS. Resource utilization remains low, requiring only a small percentage of the available FPGA logic resources, while timing analysis confirms successful timing closure with positive setup and hold margins.

The developed system demonstrates that efficient, scalable, and deterministic real-time video processing can be achieved on a low-cost FPGA platform, making it suitable for embedded vision, surveillance, robotics, and educational applications.

III.

Methodology

The design sequence is presented as a processing pipeline, where each stage performs a bounded hardware task and forwards deterministic data to the next stage.

Camera setup

The OV7670 sensor is configured through SCCB so that the pixel stream is stable and synchronous.

INPUT: sensor registers
ROLE: timing setup

Pixel capture

RGB565 data is sampled and converted into a usable internal format for downstream logic.

FORMAT: RGB565
SOURCE: OV7670

Frame buffering

Dual-port BRAM stores and forwards image data while keeping read and write domains separate.

MEMORY: BRAM
MODE: dual-port

Filter bank

Eight combinational filters evaluate the same pixel stream and deliver distinct visual outcomes.

FILTERS: 8
TYPE: hardware

VGA output

The selected result is routed to a 640 x 480 display with consistent timing and frame stability.

DISPLAY: VGA
CLOCK: 25 MHz

IX.

Conclusion

The project confirms that a compact FPGA system can acquire camera input, process it with multiple image filters, and present the result on a VGA monitor in real time. The result set and demonstration clips together provide a complete project showcase.

Possible extensions include converting the prototype into a custom silicon implementation, extending the pipeline to higher resolutions and stronger timing margins, and capturing additional controlled scenes for comparative evaluation.
